# LU18.A04 Startplan – Terminumfrage-Tool

## Abschnitt A: Architektur + Datenmodell

### Architektur (MVP)
- **Seite 1 `/`**: Umfrage erstellen
  - Komponenten: `CreatePollPage`, Eingabefelder für Titel + Optionen
- **Seite 2 `/poll/:pollId`**: Abstimmen + Resultate
  - Komponenten: `PollPage`, `VoteTable`
- **State-Strategie**
  - Lokaler Formular-State pro Seite via `useState`
  - Persistenz via `src/utils/storage.ts` (LocalStorage)
  - Kein globales State-Management nötig für MVP

### Datenmodell (TypeScript)
```ts
type VoteValue = 'yes' | 'no' | 'maybe';

interface PollOption {
  id: string;
  label: string;
}

interface Vote {
  optionId: string;
  voterName: string;
  value: VoteValue;
}

interface Poll {
  id: string;
  title: string;
  createdAt: string;
  options: PollOption[];
  votes: Vote[];
}
```

### LocalStorage-Struktur
- Key: `terminumfrage.polls.v1`
- Value: `Record<string, Poll>`
- Beispiel:
```json
{
  "a1b2c3d4": {
    "id": "a1b2c3d4",
    "title": "Teammeeting",
    "createdAt": "2026-03-10T07:00:00.000Z",
    "options": [
      { "id": "opt1", "label": "Mo 18:00" },
      { "id": "opt2", "label": "Di 19:00" }
    ],
    "votes": [
      { "optionId": "opt1", "voterName": "Lea", "value": "yes" }
    ]
  }
}
```

## Abschnitt B: Setup-Kommandos

```bash
# 1) Projekt erstellen (falls man bei Null startet)
npm create vite@latest terminumfrage-tool -- --template react-ts
cd terminumfrage-tool

# 2) Routing installieren
npm install react-router-dom

# 3) Starten
npm install
npm run dev
```
